## Executive Summary

A black screen in ModernGL + PyGame despite successful compilation and FPS output typically indicates one of two issues:

1. **Rendering Pipeline Issue**: Shaders compile, context is created, but nothing is visible
2. **Data Flow Issue**: Geometry is off-screen, matrices are wrong, or instance count is zero

The program is running (FPS output proves this), so the issue is NOT a crash or missing context.

---

## Common Causes (Ranked by Likelihood)

### 1. Missing or Incorrect Context Clearing (70% of cases)

**Problem**: No clear color set, or clear not called each frame  
**Symptom**: Black screen that updates (FPS changes)
```python
# WRONG - No clearing
while running:
    vao.render(program)
    pygame.display.flip()

# CORRECT - Clear each frame
while running:
    ctx.clear_color = (0.1, 0.1, 0.1, 1.0)  # Set once or each frame
    ctx.clear()
    vao.render(program)
    pygame.display.flip()
```

**Fix**: Always call `ctx.clear()` before rendering every frame

---

### 2. View/Projection Matrix Not Set or Incorrect (15% of cases)

**Problem**: Geometry rendered outside view frustum  
**Symptom**: Black screen even with clearing
```python
# WRONG - No matrices
program = ctx.program(vertex_shader=vs, fragment_shader=fs)
vao.render(program)

# CORRECT - Set matrices
projection = glm.perspective(glm.radians(45), aspect_ratio, 0.1, 100.0)
view = glm.lookAt(glm.vec3(0, 0, 3), glm.vec3(0, 0, 0), glm.vec3(0, 1, 0))
program['m_proj'].write(projection)
program['m_view'].write(view)
vao.render(program)
```

**Fix**: Always set projection and view matrices for 3D rendering

---

### 3. Instance Buffer Not Bound Correctly (10% of cases)

**Problem**: Instanced rendering with wrong attribute divisor  
**Symptom**: Black screen when using instancing
```python
# WRONG - No divisor
cube.buffer(instance_data, "3f", ["in_offset"])

# CORRECT - With divisor
cube.buffer(instance_data, "3f/i", ["in_offset"])  # /i means divisor=1
```

**Fix**: Use `/i` suffix in format string for instanced attributes

---

### 4. Render Call Missing or Instance Count = 0 (5% of cases)

**Problem**: `render()` called with 0 instances or wrong count  
**Symptom**: Black screen with instancing
```python
# WRONG - No instances parameter
vao.render(program)

# CORRECT - With instance count
vao.render(program, instances=instance_count)

# DEBUG - Print count
print(f"Rendering {instance_count} instances")
if instance_count == 0:
    print("ERROR: No instances to render!")
```

**Fix**: Always pass `instances=` parameter, verify count > 0

---

### 5. Buffer Data Format Mismatch (3% of cases)

**Problem**: VAO attributes don't match shader input  
**Symptom**: Black screen or validation errors
```python
# WRONG - Format doesn't match shader
# Shader: in vec4 color;
# Code: buffer(vbo, "3f", ["color"])  # Only 3 floats!

# CORRECT - Match shader inputs
# Shader: in vec3 position; in vec4 color;
# Code: buffer(vbo, "3f 4f", ["position", "color"])
```

**Fix**: Ensure format string matches shader inputs exactly

---

### 6. Fragment Shader Not Writing Output (2% of cases)

**Problem**: Fragment shader missing or wrong output  
**Symptom**: Black screen with no errors
```glsl
// WRONG - No output
#version 330
void main() {
}

// CORRECT - Must write to output
#version 330
out vec4 FragColor;
void main() {
    FragColor = vec4(1.0, 0.0, 0.0, 1.0);  // Red
}
```

**Fix**: Always have explicit `out vec4 FragColor;` and write to it

---

### 7. Camera Positioned Incorrectly or Scene Out of View (2% of cases)

**Problem**: Camera too far, too close, or wrong position  
**Symptom**: Black screen despite everything else being correct
```python
# WRONG - Camera inside geometry or too far
view = glm.lookAt(
    glm.vec3(0, 0, 0),      # Inside cube!
    glm.vec3(0, 0, 0),
    glm.vec3(0, 1, 0)
)

# CORRECT - Camera outside, looking at scene
view = glm.lookAt(
    glm.vec3(0, 0, 5),      # 5 units back
    glm.vec3(0, 0, 0),      # Look at origin
    glm.vec3(0, 1, 0)       # Up is Y
)
```

**Fix**: Position camera outside scene, adjust near/far planes

## PyGame + ModernGL Integration Issues

### Context Creation Timing (CRITICAL)
```python
# CORRECT ORDER
import pygame
import moderngl

# 1. Initialize pygame
pygame.init()

# 2. Create OpenGL-capable display
display = pygame.display.set_mode(
    (1280, 720),
    pygame.OPENGL | pygame.DOUBLEBUF  # MUST have both flags
)

# 3. THEN create ModernGL context
ctx = moderngl.create_context()

# 4. Verify context
print(f"OpenGL version: {ctx.version_code}")
print(f"GLSL version: {ctx.glsl_version}")
```

**Common Mistakes**:
- Creating context before PyGame display
- Using wrong flags (missing `DOUBLEBUF`)
- Not initializing pygame first

---

### Display Mode Flags
```python
# WRONG - Single buffering
pygame.display.set_mode((width, height), pygame.OPENGL)

# CORRECT - Double buffering required for smooth rendering
pygame.display.set_mode((width, height), pygame.OPENGL | pygame.DOUBLEBUF)
```

## Step-by-Step Debugging Checklist

### Phase 1: Verify Setup (5 minutes)

- [ ] PyGame initialized with `pygame.init()`
- [ ] Display created with `pygame.OPENGL | pygame.DOUBLEBUF`
- [ ] ModernGL context created AFTER display
- [ ] No exceptions in context creation
- [ ] `ctx.version_code` prints valid OpenGL version
```python
print(f"Context valid: {ctx is not None}")
print(f"OpenGL version: {ctx.version_code}")
print(f"Max texture units: {ctx.max_texture_units}")
```

### Phase 2: Simple Geometry Test (10 minutes)

Test with absolute minimum code:
```python
# Create simplest triangle
vertices = np.array([
    -0.5, -0.5,
     0.5, -0.5,
     0.0,  0.5,
], dtype='f4')

vbo = ctx.buffer(vertices)
vao = ctx.vertex_array(prog, [(vbo, '2f', 'position')])

# Render loop
while running:
    ctx.clear_color = (0.1, 0.1, 0.1, 1.0)
    ctx.clear()
    vao.render()
    pygame.display.flip()
```

If this shows a triangle, problem is not basic setup.

### Phase 3: Matrix Debugging (10 minutes)

- [ ] Projection matrix created
- [ ] View matrix created
- [ ] Matrices written to uniforms
- [ ] Shader uses matrices in `gl_Position`
```python
# Debug matrices
projection = glm.perspective(glm.radians(45), 16/9, 0.1, 100)
view = glm.lookAt(glm.vec3(0, 0, 3), glm.vec3(0, 0, 0), glm.vec3(0, 1, 0))

print("Projection matrix:")
print(projection)
print("\\nView matrix:")
print(view)

# Write to shader
program['m_proj'].write(projection)
program['m_view'].write(view)
```

### Phase 4: Instance Rendering Debugging (15 minutes)

For instanced rendering:

- [ ] Instance buffer created
- [ ] Buffer bound with `/i` divisor in format string
- [ ] Instance count > 0
- [ ] `render(instances=count)` called with correct count
```python
# Debug instancing
print(f"Instance buffer size: {instance_data.size} bytes")
print(f"Instance count: {num_instances}")
print(f"Attributes: {[attr.name for attr in vao.attributes]}")

# Verify attribute divisor
for attr in vao.attributes:
    if attr.name == "in_offset":
        print(f"in_offset divisor: {attr.divisor}")
```

### Phase 5: Shader Compilation Verification (5 minutes)
```python
try:
    program = ctx.program(vertex_shader=vs, fragment_shader=fs)
    print("✓ Shaders compiled successfully")
except Exception as e:
    print(f"✗ Shader error: {e}")
    # Print shader source for debugging
    print("Vertex shader:", vs)
    print("Fragment shader:", fs)
```

### Phase 6: Rendering State Verification (10 minutes)
```python
# Check OpenGL state
print(f"Clear color: {ctx.clear_color}")
print(f"Depth test enabled: {ctx.depth_mask}")
print(f"Wireframe mode: {ctx.wireframe}")
print(f"Cull face enabled: {ctx.cull_face}")

# Test with minimal state
ctx.disable(moderngl.DEPTH_TEST)
ctx.disable(moderngl.CULL_FACE)
```

### 3D Cube with Matrices
```python
import moderngl
import pygame
import numpy as np
import glm
import math

pygame.init()
display = pygame.display.set_mode((800, 600), pygame.OPENGL | pygame.DOUBLEBUF)
ctx = moderngl.create_context()

vertex_shader = """
#version 330
in vec3 position;
in vec3 normal;

uniform mat4 m_proj;
uniform mat4 m_view;
uniform mat4 m_model;
uniform mat3 m_normal;

out vec3 v_normal;
out vec3 v_position;

void main() {
    v_position = vec3(m_model * vec4(position, 1.0));
    v_normal = normalize(m_normal * normal);
    gl_Position = m_proj * m_view * vec4(v_position, 1.0);
}
"""

fragment_shader = """
#version 330
in vec3 v_normal;
in vec3 v_position;

out vec4 FragColor;

void main() {
    vec3 light_dir = normalize(vec3(1.0, 1.0, 1.0));
    float brightness = max(dot(v_normal, light_dir), 0.0);
    FragColor = vec4(vec3(brightness), 1.0);
}
"""

program = ctx.program(vertex_shader=vertex_shader, fragment_shader=fragment_shader)

# Create cube vertices (simplified)
vertices = np.array([
    -1, -1, -1,  0, 0, -1,
     1, -1, -1,  0, 0, -1,
     1,  1, -1,  0, 0, -1,
    -1,  1, -1,  0, 0, -1,
], dtype='f4')

vbo = ctx.buffer(vertices)
vao = ctx.vertex_array(program, [(vbo, '3f 3f', 'position', 'normal')])

# Matrices
aspect = 800 / 600
projection = glm.perspective(glm.radians(45), aspect, 0.1, 100)
view = glm.lookAt(
    glm.vec3(3, 3, 3),
    glm.vec3(0, 0, 0),
    glm.vec3(0, 1, 0)
)
model = glm.mat4(1.0)

program['m_proj'].write(projection)
program['m_view'].write(view)
program['m_model'].write(model)

# Calculate normal matrix
normal_matrix = glm.transpose(glm.inverse(glm.mat3(model)))
program['m_normal'].write(normal_matrix.to_bytes())

# Render
ctx.enable(moderngl.DEPTH_TEST)
ctx.clear_color = (0.1, 0.1, 0.1, 1.0)

running = True
while running:
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False
    
    ctx.clear()
    vao.render()
    pygame.display.flip()

pygame.quit()
```

## Working Voxel Implementation

Based on ModernGL's official `voxel_cubes.py` example:
```python
from array import array
import glm
import moderngl

class Voxel:
    """Simple voxel cube renderer using GPU instancing"""
    
    def __init__(self, ctx, size=(100, 100, 100)):
        self.ctx = ctx
        self._size = size
        
        # 1. Create lookup texture (determines which voxels are active)
        self.voxel_lookup = ctx.texture((1000, 1000), 1, dtype='f1')
        self.voxel_lookup.filter = moderngl.NEAREST, moderngl.NEAREST
        self.voxel_lookup.repeat_x = False
        self.voxel_lookup.repeat_y = False
        
        # Fill with default data (all active)
        for i in range(100):
            self.fill_layer(i, 255)
        
        # 2. Create buffer for per-instance data
        self.instance_data = ctx.buffer(reserve=self.max_cubes * 4 * 3)
        
        # 3. Get cube geometry
        from moderngl_window import geometry
        self.cube = geometry.cube()
        
        # 4. Bind instance buffer with divisor
        # Format "3f/i" = 3 floats per instance with divisor 1
        self.cube.buffer(self.instance_data, "3f/i", ["in_offset"])
        
        # 5. VAO for transform feedback (generates instance data)
        self.gen_instance_vao = None
        self._query = ctx.query(primitives=True)
        self._num_instances = 0
        
        # Will be set externally
        self.gen_instance_prog = None
        self.voxel_light_prog = None
    
    @property
    def max_cubes(self):
        return self._size[0] * self._size[1] * self._size[2]
    
    def rebuild(self):
        """Generate instance data using transform feedback"""
        if not self.gen_instance_vao:
            # Create VAO with no input buffers (for transform feedback)
            self.gen_instance_vao = self.ctx.vertex_array(
                self.gen_instance_prog, []
            )
        
        # Set uniforms
        self.gen_instance_prog['voxel_size'] = self._size
        self.voxel_lookup.use(location=0)
        
        # Run transform feedback
        with self._query:
            self.gen_instance_vao.transform(
                self.instance_data,
                mode=moderngl.POINTS,
                vertices=self.max_cubes
            )
        
        # Get actual instance count
        self._num_instances = self._query.primitives
        print(f"Active voxels: {self._num_instances}")
    
    def render(self, projection_matrix, camera_matrix):
        """Render voxels with instancing"""
        # Calculate transformation
        translate = glm.translate(glm.vec3(
            -self._size[0] / 2,
            -self._size[1] / 2,
            -self._size[2] * 2
        ))
        mat = camera_matrix * translate
        
        # Set uniforms
        self.voxel_light_prog['m_proj'].write(projection_matrix)
        self.voxel_light_prog['m_modelview'].write(mat)
        
        # CRITICAL: Pass instance count
        self.cube.render(self.voxel_light_prog, instances=self._num_instances)
    
    def fill_layer(self, layer, value):
        """Update a layer in the lookup texture"""
        x = (layer % 10) * self._size[0]
        y = (layer // 10) * self._size[1]
        self.voxel_lookup.write(
            array('B', [value] * 100 * 100),
            viewport=(x, y, 100, 100)
        )
```

## Buffer and VAO Setup

### Format String Reference
```python
# Basic formats (non-instanced)
"3f"           # 3 floats
"3f 4f"        # 3 floats, 4 floats
"2f 1i"        # 2 floats, 1 int

# Instanced formats (with /i divisor)
"3f/i"         # 3 floats, divisor 1
"4f 4f/i"      # 4 floats, 4 floats with divisor on second
"3f/i 1f/i"    # Both with divisor

# Meaning of /i
# The /i suffix means the attribute has divisor=1
# Divisor=0: advances per vertex (default)
# Divisor=1: advances per instance
```

### Common Buffer Mistakes
```python
# MISTAKE 1: Wrong format
# Shader: in vec3 position;
# Code: vbo = ctx.buffer(vertices)
#       vao = ctx.vertex_array(prog, [(vbo, '2f', 'position')])  # Only 2 floats!

# FIX: Ensure format matches data
vertices = np.array([-0.5, -0.5, 0, 0.5, -0.5, 0, ...], dtype='f4')  # 3 floats per vertex
vao = ctx.vertex_array(prog, [(vbo, '3f', 'position')])  # 3 floats

# MISTAKE 2: Missing divisor for instancing
# Code: cube.buffer(instance_data, "3f", ["in_offset"])  # No /i!

# FIX: Add divisor
cube.buffer(instance_data, "3f/i", ["in_offset"])  # Now has divisor

# MISTAKE 3: Attribute not in shader
# Shader: in vec3 position;
# Code: vao = ctx.vertex_array(prog, [(vbo, '3f', 'color')])  # Wrong name!

# FIX: Match shader attribute names exactly
vao = ctx.vertex_array(prog, [(vbo, '3f', 'position')])  # Must match
```

## Matrix and Camera Debugging

### Projection Matrix
```python
import glm

# Create projection matrix
fov = 45.0              # Field of view in degrees
aspect = 800 / 600      # Width / height
near = 0.1             # Near clipping plane
far = 100.0            # Far clipping plane

projection = glm.perspective(
    glm.radians(fov),
    aspect,
    near,
    far
)

# Debug: print matrix
print("Projection matrix:")
for i in range(4):
    print([projection[j][i] for j in range(4)])

# Common issues:
# - Near plane too large: geometry clipped off
# - Far plane too small: geometry clipped off
# - Wrong aspect ratio: stretched/squashed geometry
# - FOV too small: objects appear far away
```

### View Matrix (Camera)
```python
# Create view matrix with lookAt
eye = glm.vec3(3, 3, 3)        # Camera position
center = glm.vec3(0, 0, 0)     # Look at point
up = glm.vec3(0, 1, 0)         # Up direction

view = glm.lookAt(eye, center, up)

# Debug camera position
print(f"Camera at: {eye}")
print(f"Looking at: {center}")

# Common issues:
# - Camera inside geometry: nothing visible
# - Camera too far: geometry appears tiny
# - Camera behind geometry: nothing visible
# - Up vector parallel to view direction: undefined behavior
```

### Model Matrix
```python
# Create model matrix
model = glm.mat4(1.0)  # Identity (no transformation)

# Apply transformations
model = glm.translate(model, glm.vec3(1, 0, 0))  # Move
model = glm.rotate(model, glm.radians(45), glm.vec3(0, 1, 0))  # Rotate
model = glm.scale(model, glm.vec3(0.5, 0.5, 0.5))  # Scale

# Combine into MVP matrix
```

### Complete Camera Setup Example
```python
def setup_camera(ctx, program, window_width, window_height):
    """Setup projection and view matrices"""
    
    # Projection matrix
    aspect = window_width / window_height
    projection = glm.perspective(
        glm.radians(60),    # FOV
        aspect,
        0.1,                # Near
        1000.0              # Far - increase for large scenes
    )
    
    # View matrix
    view = glm.lookAt(
        glm.vec3(0, 5, 10),     # Camera position
        glm.vec3(0, 0, 0),      # Look at center
        glm.vec3(0, 1, 0)       # Up vector
    )
    
    # Model matrix
    model = glm.mat4(1.0)
    
    # Send to shader
    program['m_proj'].write(projection)
    program['m_view'].write(view)
    program['m_model'].write(model)
    
    # Optional: send combined MVP
    mvp = projection * view * model
    program['m_mvp'].write(mvp)
```

## Shader Debugging Techniques

### Compile and Check for Errors
```python
def compile_and_check(ctx, vertex_src, fragment_src):
    """Compile shaders and catch errors"""
    try:
        program = ctx.program(
            vertex_shader=vertex_src,
            fragment_shader=fragment_src
        )
        print("✓ Shaders compiled successfully")
        return program
    except Exception as e:
        print(f"✗ Shader compilation error:")
        print(f"  {e}")
        
        # Print source for debugging
        print("\\nVertex shader:")
        print(vertex_src)
        print("\\nFragment shader:")
        print(fragment_src)
        
        return None
```

### Verify Uniforms Are Set
```python
def set_uniforms_safe(program, uniforms):
    """Safely set uniforms with error checking"""
    for name, value in uniforms.items():
        try:
            program[name].write(value)
            print(f"✓ Set uniform '{name}'")
        except KeyError:
            print(f"✗ Uniform '{name}' not found in shader")
        except Exception as e:
            print(f"✗ Error setting '{name}': {e}")
```

```python
projection = glm.perspective(
```

